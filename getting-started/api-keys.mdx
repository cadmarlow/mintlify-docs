## API Key #1: Firecrawl

**Used for**: Web scraping, prospect search, domain analysis

### Step 1: Create Firecrawl Account

1. Go to [firecrawl.dev](https://firecrawl.dev)
2. Click **"Sign Up"**
3. Choose **"Developer"** plan:
   - **Free tier**: 500 credits/month
   - **Starter**: $20/month for 5,000 credits
   - **Pro**: $100/month for 50,000 credits

<Tip>
Start with the free tier (500 credits = ~100 prospect searches/month).
</Tip>

### Step 2: Get API Key

1. Go to [Dashboard](https://firecrawl.dev/app/dashboard)
2. Click **"API Keys"** in sidebar
3. Click **"Create API Key"**
4. Copy the key (starts with `fc-`)

<Frame>
  <img src="/images/firecrawl-api-key.png" alt="Firecrawl API key" />
</Frame>

### Step 3: Add to tldr.work

1. Go to tldr.work **Settings** > **API Keys**
2. Paste key in **"Firecrawl API Key"** field
3. Click **"Test Connection"** (optional but recommended)
4. Click **"Save"**

<Check>
‚úÖ Firecrawl configured! Prospect searches now use your key.
</Check>

## API Key #2: Hunter.io

**Used for**: Email finding and verification

### Step 1: Create Hunter Account

1. Go to [hunter.io](https://hunter.io)

<Info>

### Step 2: Get API Key

## API Key #3: OpenRouter

**Used for**: AI queries (ChatGPT, Claude, Gemini, Perplexity, Mistral), email generation, domain analysis, prompt generation

1. Go to [openrouter.ai](https://openrouter.ai)

## API Key #4: Resend

**Used for**: Sending campaign emails

### Step 1: Create Resend Account

1. Go to [resend.com](https://resend.com)
2. Click **"Start Building"**
3. Sign up with email/GitHub
4. Choose a plan:
   - **Free**: 3,000 emails/month
   - **Pro**: $20/month for 50,000 emails
   - **Enterprise**: Custom pricing

<Info>
Free tier is perfect for testing! Upgrade when you hit limits.
</Info>

### Step 2: Verify Domain (Optional but Recommended)

For best deliverability, verify your sending domain:

1. Go to [Domains](https://resend.com/domains)
2. Click **"Add Domain"**
3. Add your domain (e.g., `acme.com`)
4. Add DNS records (SPF, DKIM, DMARC) to your DNS provider
5. Wait for verification (~24 hours)

<Warning>
Unverified domains can send emails but may have lower deliverability (more spam flags).
</Warning>

### Step 3: Get API Key

1. Go to [API Keys](https://resend.com/api-keys)
2. Click **"Create API Key"**
3. Name it: "tldr.work"
4. Copy the key (starts with `re_`)

<Frame>
  <img src="/images/resend-api-key.png" alt="Resend API key" />
</Frame>

### Step 4: Add to tldr.work

1. Go to tldr.work **Settings** > **API Keys**
2. Paste key in **"Resend API Key"** field
3. Enter verified domain (if you did Step 2)
4. Click **"Test Connection"** (sends test email)
5. Click **"Save"**

<Check>
‚úÖ Resend configured! Campaign emails now use your key and domain.
</Check>

## Managing API Keys

### Viewing Current Keys

Go to **Settings** > **API Keys**

You'll see:
- ‚úÖ **Configured**: Key added and working
- ‚ö†Ô∏è **Not configured**: Using shared key
- ‚ùå **Error**: Key invalid or no credits

### Removing Keys

To remove a key (revert to shared keys):
1. Click **"Remove"** next to the key
2. Confirm deletion
3. tldr.work will use shared keys again

<Warning>
Removing keys switches back to shared pricing (higher costs + 40% margin).
</Warning>

### Updating Keys

If your key expires or you want to use a different account:
1. Paste new key in the same field
2. Click **"Test Connection"**
3. Click **"Save"**

Old key is replaced immediately.

## Security Best Practices

<AccordionGroup>
  <Accordion title="Keep Keys Private" icon="lock">
    - Never share API keys in emails, chat, or screenshots
    - Don't commit keys to git repositories
    - Use environment variables in code (if self-hosting)
  </Accordion>

  <Accordion title="Use Dedicated Keys" icon="key">
    - Create separate API keys for tldr.work (don't reuse keys from other tools)
    - Name keys clearly: "tldr.work Production"
    - Easier to rotate or revoke if compromised
  </Accordion>

  <Accordion title="Monitor Usage" icon="chart-line">
    - Check each service's dashboard monthly
    - Set up spending alerts (OpenRouter, Resend)
    - Watch for unusual activity (sign of key compromise)
  </Accordion>

  <Accordion title="Rotate Keys Regularly" icon="rotate">
    - Change keys every 6-12 months
    - Immediately rotate if you suspect compromise
    - Update in tldr.work Settings > API Keys
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: API key is invalid">
    **Solutions**:
    - Copy key again (no extra spaces or characters)
    - Check if key was deleted in the service's dashboard
    - Create a new key and try again
  </Accordion>

  <Accordion title="Error: Insufficient credits">
    **For OpenRouter**: Add credits at [openrouter.ai/credits](https://openrouter.ai/credits)  
    **For Firecrawl**: Upgrade plan at [firecrawl.dev/pricing](https://firecrawl.dev/pricing)  
    **For Hunter**: Upgrade plan at [hunter.io/pricing](https://hunter.io/pricing)
  </Accordion>

  <Accordion title="Email deliverability is low">
    **For Resend**:
    1. Verify your sending domain (add SPF, DKIM, DMARC)
    2. Warm up your domain (start with 20-30 emails/day, increase gradually)
    3. Avoid spam triggers (no ALL CAPS, excessive punctuation, spammy words)
    4. Check [Resend logs](https://resend.com/logs) for bounce reasons
  </Accordion>

  <Accordion title="Test connection fails but key is correct">
    **Possible causes**:
    - Service is temporarily down (check status page)
    - API rate limit exceeded (wait a few minutes)
    - Firewall blocking API requests (rare)
    
    **Solution**: Wait 5 minutes and try again. Contact support if persists.
  </Accordion>
</AccordionGroup>

## Cost Optimization Tips

1. **Start with free tiers**: Test each service before committing to paid plans

2. **Monitor usage weekly**: Catch overages early before they get expensive

3. **Use cheaper models when possible**:
   - Domain analysis: GPT-5-mini instead of GPT-5 (same quality, 10x cheaper)
   - Simple emails: GPT-4o-mini (good enough for most cases)

4. **Batch operations**: Search for 50 prospects at once (more efficient than 50 individual searches)

5. **Disable unused engines**: If you only care about ChatGPT visibility, don't test on all 5 engines
